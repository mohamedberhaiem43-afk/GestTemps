## 8. Sujets de conformité particuliers

### 8.1 App Privacy (Apple) — données collectées à déclarer

Cocher exactement ces cases dans App Store Connect → App Privacy → Data Types :

- **Contact Info** : Name, Email Address, Phone Number → Used for: App Functionality, Personalization
- **Health & Fitness** : aucun
- **Financial Info** : Payment Info → Used for: App Functionality (via Stripe, non stocké chez nous)
- **Location** : Precise Location → Used for: App Functionality — **seulement si GeoZones activé par le tenant** (cocher uniquement si vous prévoyez de l'activer en prod)
- **Sensitive Info** : aucun
- **Contacts** : aucun
- **User Content** : Photos or Videos, Other User Content → Used for: App Functionality
- **Browsing History** : aucun
- **Search History** : aucun
- **Identifiers** : User ID, Device ID → Used for: App Functionality, Analytics
- **Purchases** : Purchase History → Used for: App Functionality
- **Usage Data** : Product Interaction, Other Usage Data → Used for: App Functionality, Analytics
- **Diagnostics** : Crash Data, Performance Data, Other Diagnostic Data → Used for: App Functionality, Analytics

Toutes ces données sont liées à l'identité de l'utilisateur (linked to the user) — pas anonymes.
Aucune n'est utilisée pour du tracking publicitaire.

### 8.2 Data Safety (Google Play) — équivalent du Privacy Nutrition Label

Données collectées (cocher dans Play Console → App content → Data safety) :

| Catégorie | Type | Collected | Shared | Required | Purpose |
|---|---|---|---|---|---|
| Personal info | Name, Email, Phone | Yes | No | Yes | App functionality, Account management |
| Financial info | Payment info | Yes | No | Yes | Purchases (via Stripe) |
| Location | Approximate / Precise | Conditional | No | No | Attendance (only if GeoZones enabled by tenant) |
| Photos and videos | Photos | Yes | No | No | App functionality (expense receipts) |
| Files and docs | Files | Yes | No | No | App functionality (signed contracts, payslips) |
| App activity | App interactions, Other actions | Yes | No | Yes | Analytics, App functionality |
| App info and performance | Crash logs, Diagnostics | Yes | No | No | App functionality |
| Device or other IDs | Device ID | Yes | No | Yes | App functionality, Fraud prevention |

Security practices à déclarer :
- ✅ Data is encrypted in transit
- ✅ You can request that data be deleted
- ✅ Committed to Play Families Policy: No (B2B, not for children)

### 8.3 Export Compliance (Apple)

Réponse au prompt sur l'usage du chiffrement : **Yes**, mais utilisation exclusivement
de standards (HTTPS, biométrie OS). Sélectionner :
- "Your app uses encryption" → Yes
- "Does your app qualify for any exemptions?" → Yes (5D002)
- "Standard cryptographic algorithms" → Yes

→ Déjà géré côté config via `ITSAppUsesNonExemptEncryption: false` dans `app.json` ;
le prompt ne se redéclenchera plus à chaque build.
